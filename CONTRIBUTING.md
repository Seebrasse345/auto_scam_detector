## ğŸš€ How to Contribute

### 1. Working on Assigned Tasks

If you're assigned to a task:
1. Fork this repo
2. Create a branch: `git checkout -b task-1-patterns` (or your task)
3. Complete your deliverable
4. Test thoroughly
5. Submit a PR with:
   - Task number in title
   - Code/files
   - Test results
   - Any issues encountered

### 2. Contributing New Ideas

Have an idea for improvement?
1. Open an Issue describing your idea
2. Wait for discussion/approval
3. Fork and create a branch
4. Implement
5. Submit PR

### 3. Reporting Bugs

Found a bug?
1. Check existing Issues
2. If new, create an Issue with:
   - Clear description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details

### 4. Documenting Threats

Spotted a new threat?
1. Document it in an Issue
2. Include:
   - Threat type
   - Evidence (post ID, username, content)
   - Why it's a threat
   - Pattern to detect it
3. We'll add it to [THREATS.md](./THREATS.md)
